# Chat Example

This example shows the goldr-friendly SSE pattern:

- regular generated actions receive user input
- an app-owned `GET /chat/events` handler streams HTML over SSE
- the `sse` package handles event-stream headers, comments, data fields, and
  templ-rendered named HTML events
- the `browser` package serves Goldr's named SSE event helper from `/goldr/`
- messages live in process memory
- HTMX stays visible in templates

Run:

```bash
go run ./examples/chat
```

Then open the printed localhost URL in a browser. Enter a name, open a second
browser window with another name, and send messages. The send action waits for
three seconds on the server so the HTMX progress indicator is visible.

Useful paths:

```text
/
/join
/chat
/chat/message
/chat/sign-out
/chat/events
/goldr/goldr-sse-event.js
```

`/chat/events` is intentionally registered in `main.go`, not generated by
goldr. The example keeps SSE as application-owned server behavior while pages,
actions, layouts, and URL helpers stay generated from `app/routes`. The stream
handler owns subscribers and replay behavior; `sse` only writes valid SSE
responses. The stream sends named `chat-message` events, and the template uses
`goldr-sse-event="chat-message"` so htmx swaps only that named event.

Regenerate goldr-owned files from the repository root:

```bash
go run ./cmd/goldr generate --root examples/chat
```

Check generated files without writing:

```bash
go run ./cmd/goldr check --root examples/chat
```
